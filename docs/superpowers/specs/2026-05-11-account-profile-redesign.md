# 账号档案面板重新设计

## 背景

将账号设置页的「档案」标签页从原有的角色卡片 + 进度条 + 简介布局，重新设计为更紧凑的 Galgame 风格成员档案卡。去除进度条和简介，增加同好会专属展示区域和个性签名。

## 设计目标

- 粉色 Galgame 可爱风 → 回退到站点现有红白配色
- 去除进度条（同好会/刊物/活动）
- 去除简介框（bio）
- 去除 emoji 装饰和成就徽章
- 同好会改为专属框展示，支持多同好会
- 同好会条目显示圆形头像
- 角色标签使用对应颜色
- 个性签名可编辑

## 角色颜色对应

| 角色 | 颜色 | 色值 |
|------|------|------|
| 访客 (visitor) | 灰色 | #888 |
| 成员 (member) | 绿色 | #4caf50 |
| 管理员 (manager) | 蓝色 | #2196f3 |
| 会长 (representative) | 橙色 | #ff9800 |
| 超级管理员 (super_admin) | 粉色 | #e91e63 |

## 配色方案

使用站点 CSS 变量中的颜色：
- `--md-primary: #e74c3c` — 主色，用于头像背景、数字、称号
- `--md-surface-container: #f8f8f8` — 区块背景
- `--md-outline-variant: #e0e0e0` — 边框
- 白色 (#fff) — 卡片底色

## 后端改动

### users 表新增字段
- `profile_bio` VARCHAR(300) DEFAULT '' — 个性签名

### API 端点
- `api/auth.php?action=update_profile` — POST 更新 profile_bio
- `api/auth.php?action=me` — GET 返回 profile_bio（合并到现有 me 接口）

### 收集数据来源
- 同好会数据：`club_memberships` 表（已有）
- 统计数据：刊物投稿数、报名活动数、活跃天数
- 刊物投稿数：`publications` 表按 user_id 统计
- 活跃天数：暂设为 0（待后续实现）

## 前端改动

### index.html
- 重构 `#vnProfileTab` 内容为新的布局结构
- 去掉 progress 条、bio 框的 HTML
- 添加个性签名显示区域
- 同好会列表保持现有 DOM 结构但样式更新
- settings tab 添加个性签名编辑入口

### js/app.js
- `renderVNProfile()` 重写：
  - 头部：头像 + 名字 + 称号 + 签名
  - 同好会区域：遍历 memberships，显示头像、名称、时间、角色标签
  - 统计：从现有数据计算
  - 底部：加入时间 + 活跃时间
- 同好会头像：从 `State.bandoriRows` / `State.japanRows` 中查找 club 的 `logo_url`
- 新增 `updateProfileBio()` 函数
- 设置页添加个性签名编辑 UI 和保存逻辑

### css/styles.css
- 新的 `.vn-profile-card` 样式取代 `.vn-character-card`
- 同好会条目样式：头像圆形、角色标签颜色
- 统计卡片样式
- 响应式适配
