# 顶层用户信息框设计方案

## 背景

当前页面布局中，用户登录入口和同好会导航按钮分散在左侧卡片（introCard）和右侧侧边按钮中，缺少统一的用户操作入口。本次改造新增一个顶层用户信息框，将导航、用户信息和管理入口聚合到一个组件中。

## 设计目标

1. 提供统一的用户信息展示和导航操作入口
2. 桌面端和移动端各自适配布局
3. 保留现有左侧 introCard 和右侧 selectedCard 主体内容
4. 移除右侧五个侧边导航按钮

## 桌面端设计

### 位置与样式

- 顶部居中定位：`position: absolute; top: 8px; left: 50%; transform: translateX(-50%)`
- 毛玻璃效果：`backdrop-filter: blur(16px)`
- 圆角 20px，宽度自适应（约 390-460px）
- 与现有 md3-card 视觉风格一致

### 内容结构

```
┌─────────────────────────────────────────┐
│ [头像] 用户名 [管理员]    [账号] [管理] │  ← 用户行
│─────────────────────────────────────────│
│ 中国 │ 日本 │ 海外 │ 日历 │ 刊物        │  ← 导航按钮行（flex:1 等宽）
└─────────────────────────────────────────┘
```

**用户行：**
- 未登录：显示"访客" + 登录/注册按钮（紫色填充）
- 已登录：显示用户头像 + 用户名 + 角色标签 + 账号按钮 + 同好会管理按钮
- 同好会管理按钮仅管理员（`hasRole('manager')`）可见，粉色渐变

**导航按钮行：**
- 五个按钮（中国/日本/海外/日历/刊物）同一行等宽分布
- 点击触发已有功能（`switchToChinaMap()` 等）
- 按钮高度约 28px，字号 11px

## 移动端设计

### 折叠/展开交互

移动端顶部用户区采用可折叠设计：

- **折叠态**：仅显示用户行 + 展开箭头（▼），占用约 48px
- **展开态**：显示用户行 + 2列网格导航按钮，管理员额外显示管理按钮
- 点击用户行切换折叠/展开状态，带动画过渡
- 刊物投稿按钮占 grid 整行（span 2）

### 左侧隐藏菜单

- 左上角汉堡按钮（☰）点击展开左侧抽屉
- 抽屉内含原有 introCard 内容：logo、简介、主题/反色开关
- 覆盖层样式，点击遮罩或关闭按钮收起

### 文件结构

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `index.html` | 新增 | 顶部卡片 HTML 结构、移动端左侧抽屉容器 |
| `css/styles.css` | 新增 | 顶部卡片样式、移动端折叠/展开、左侧抽屉样式 |
| `js/app.js` | 修改 | 添加顶部卡片交互逻辑、移除侧边按钮事件绑定 |

### 交互说明

| 元素 | 交互 |
|------|------|
| 导航按钮（中国同好会等） | 点击触发现有 `switchToChinaMap()` 等函数 |
| 登录/注册按钮 | 点击打开 `#accountModal`（登录视图） |
| 账号按钮 | 点击打开 `#accountModal`（设置视图） |
| 同好会管理按钮 | 点击跳转 `admin/club_manager.html` |
| 移动端展开箭头 | 点击切换折叠/展开状态 |
| 移动端汉堡按钮 | 点击打开左侧抽屉菜单 |

### 注意事项

- 所有按钮复用现有功能，不改变原有业务逻辑
- 顶部卡片不干扰地图交互（`pointer-events` 穿透处理）
- 管理员按钮仅在 `hasRole('manager')` 时显示
- 移动端折叠状态由 CSS class 控制，JS 切换 `.collapsed` / `.expanded`
